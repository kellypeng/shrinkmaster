Features

Six things, done well.

Instead of fifty things, half-baked. (Looking at you, every other "all-in-one" file utility.)

100% local

Your videos never leave your Mac. No upload, no cloud, no account. The only network call ShrinkMaster makes is to GitHub Releases — and only when you click "check for updates."

Hardware accelerated

Auto-detects VideoToolbox on Mac, NVENC / Quick Sync / AMF on PC. CPU fallback uses SVT-AV1 or libx265 — still fast.

H.265 + AV1

Modern codecs with sane defaults. H.265 is fast and universally compatible. AV1 trades speed for the smallest file size you can ship.

Drag, drop, batch.

One file or a hundred. Real cancel during compression — actually kills ffmpeg, cleans up half-written output. Save As lands the result anywhere.

EN + 中文

Bilingual from day one — follows your system language by default, switchable in Advanced Settings. Most ffmpeg GUIs are English-only.

Free forever

MIT licensed. Source on GitHub. No paid tier, no in-app purchase, no telemetry. If I disappear tomorrow you can fork it and keep going.

Opening on macOS for the first time

ShrinkMaster isn't yet code-signed (Apple Developer Program is $99/yr — a real expense for a free tool). macOS will block the first launch with "cannot be opened because the developer cannot be verified." Bypass takes 5 seconds, only needed once.

  1. Drag ShrinkMaster.app into your Applications folder
  2. Right-click the app → choose Open (don't double-click yet)
  3. Click Open in the confirmation dialog macOS shows

After that, it opens like any other app. If macOS still refuses, run this in Terminal:

xattr -cr /Applications/ShrinkMaster.app

FAQ

Things you might be wondering.

Is it really free? Forever?

Yes. MIT licensed. No paid tier, no in-app purchase, no upsell. Source is public on GitHub — if I vanish, you can fork it.

If you want to thank me: a Star on GitHub or a follow on Twitter is plenty.

Why does it only do videos?

Because scope discipline is what makes a tool actually good. Compressing images well needs different libraries than video. PDFs need different libraries again. GIFs are essentially a third thing.

Apps that try to compress "everything" are usually mediocre at all of it. ShrinkMaster picked one job. If you need image / PDF compression there are better dedicated tools — I'm happy to recommend some on Twitter.

Why isn't the app code-signed? Is it dangerous?

Apple Developer Program is $99/year — a real expense for a free tool with no revenue. So v0.1 ships unsigned ad-hoc, and you bypass Gatekeeper once via right-click → Open.

Source is fully public. Audit every line if you want, build it yourself, or have your favorite LLM read it for you.

Does it phone home? Send any data?

No analytics, no telemetry, no crash reporting. The only network call is to GitHub Releases — and only when you explicitly click "Check for updates." By default the app is offline.

Your videos never leave your machine. No temp files in the cloud. Nothing.

Why is the DMG so big (~300 MB)?

Because the app bundles its own ffmpeg + ffprobe binaries (~130 MB each, universal2). That makes it work on any Mac immediately — no Homebrew, no command-line setup, just drag and run. Apple Silicon and Intel both run native code, no Rosetta needed.

How is this different from HandBrake?

HandBrake is more powerful and you should keep it for advanced workflows. ShrinkMaster does one thing — make a video smaller — with sane defaults, less surface area, and bilingual UI. If your workflow is "drop a video, get a smaller one," ShrinkMaster is faster to use. If you need a custom encoder ladder, use HandBrake or ffmpeg directly.

Will there be a Windows version?

The codebase is platform-neutral — it's just Electron + ffmpeg subprocess. A Windows build is planned. Star the repo to get notified.
