Features

Built for getting it done

Six things, well — instead of fifty things, badly.

100% local

Your videos never leave your Mac. No upload, no cloud, no account. The app makes one optional GitHub call only when you click "check for updates."

Hardware accelerated

Auto-detects VideoToolbox on Mac, NVENC / Quick Sync / AMF on PC. Falls back to a fast SVT-AV1 / libx265 CPU path if no GPU encoder is available.

H.265 + AV1

Modern codecs with sane defaults. The H.265 preset is fast and universally compatible; AV1 trades speed for the smallest file size.

Drag & drop · batch

Drop one file or a hundred. Real cancel during compression — kills the ffmpeg process and cleans up. Save As dialog routes the result anywhere.

Bilingual

English and 简体中文 — follows your system language by default. Switch any time in Advanced Settings. The first ffmpeg GUI built bilingual from day one.

Free forever

MIT licensed, source on GitHub, no in-app purchases, no upsells, no telemetry. If it dies, you can fork it.

Opening on macOS for the first time

ShrinkMaster isn't yet signed with an Apple Developer ID, so macOS will block the first launch with "cannot be opened because the developer cannot be verified." Workaround takes 5 seconds, only needed once.

  1. Drag ShrinkMaster.app into your Applications folder
  2. Right-click the app → choose Open (don't double-click yet)
  3. Click Open in the confirmation dialog macOS shows

After that, it opens normally. If macOS still refuses, run this in Terminal:

xattr -cr /Applications/ShrinkMaster.app

FAQ

Frequently asked questions

Is it really free? Forever?

Yes. MIT licensed. No paid tier, no in-app purchases, no upsells. The source is public on GitHub — if I ever vanish, you can fork it.

If you want to thank me, a Star on GitHub or a follow on Twitter is plenty.

Why is the app not code-signed? Is it dangerous?

Apple Developer Program membership costs $99/year — a real expense for a free tool with no revenue. So v0.1.0 ships unsigned ad-hoc, and you bypass Gatekeeper once via right-click → Open.

The source code is fully public. You can audit every line, build it yourself, or have your favorite LLM read it for you.

Does the app phone home or send any data?

No analytics, no telemetry, no crash reporting. The only network call ShrinkMaster ever makes is to the GitHub Releases API — and only when you explicitly click "Check for updates" in Advanced Settings. By default the app is offline.

Your videos never leave your machine. No upload, no temp files in the cloud, nothing.

Why is the DMG so big (~300 MB)?

Because the app bundles its own ffmpeg + ffprobe binaries (about 130 MB each, universal2). That makes it work on any Mac immediately — no Homebrew, no command-line setup, just drag and run. Apple Silicon and Intel both run native code; no Rosetta needed.

Will there be a Windows or Linux version?

The codebase is platform-neutral — it's just Electron + ffmpeg subprocess. A Windows build is planned and tracked in GitHub Issues. Star the repo to be notified.

How is this different from HandBrake?

HandBrake is more powerful and you should keep it for advanced workflows. ShrinkMaster does one thing — shrink a video — with sane defaults, less surface area, and bilingual UI. If your workflow is "drop a video, get a smaller one," ShrinkMaster is faster to use. If you need to script your encoder ladder, use HandBrake or ffmpeg directly.

Can I use it for commercial work?

Yes. The app is MIT licensed; the bundled ffmpeg uses GPL-licensed encoders. As long as you're using the app to compress your own videos, you're fine. If you're embedding ShrinkMaster's code in another product you ship, the GPL terms of the bundled ffmpeg apply — at which point you should use a different ffmpeg build or talk to a lawyer.
